# Get Prayer Times

Here are some examples of how to get daily prayer times and monthly prayer times.

In these examples, we will use Semarang as the location. Semarang has the `id` of `1433`. You can find the list that shows the `KOTA SEMARANG` id here: <Link to={"/docs/locations#location-81-wonosobo-regency---location-90-kulon-progo-regency"}>`KOTA SEMARANG` id</Link> page.

## Daily Prayer Times

To get the daily prayer times, we are using this endpoint. I have compiled the list of all locations ID can be found in <Link to={"/docs/locations"}>Location</Link> page.

```http title="HTTP GET"
https://api.myquran.com/v2/sholat/jadwal/{location-id}/{date}
```

:::info `date` format

API Muslim v2 accept these formats for the `date` parameter. Feel free to choose one of these formats:

- `yyyy/mm/dd`
  - Example: `2025/06/26`
- `yyyy-mm-dd`
  - Example: `2025-06-26`

:::

### Example of fetching daily prayer times using `yyyy/mm/dd` date format

```http title="Daily Prayer Times for Semarang on 26 June 2025 (June 26, 2025)"
https://api.myquran.com/v2/sholat/jadwal/1433/2025/06/26
```

```json title="Response"
{
	"status": true,
	"request": {
		"path": "/sholat/jadwal/1433/2025/06/26"
	},
	"data": {
		"id": 1433,
		"lokasi": "KOTA SEMARANG",
		"daerah": "JAWA TENGAH",
		"jadwal": {
			"tanggal": "Kamis, 26/06/2025",
			"imsak": "04:18",
			"subuh": "04:28",
			"terbit": "05:46",
			"dhuha": "06:16",
			"dzuhur": "11:45",
			"ashar": "15:05",
			"maghrib": "17:36",
			"isya": "18:50",
			"date": "2025-06-26"
		}
	}
}
```

:::info 24-hour Format

In API Muslim v2, prayer times use the 24-hour format, as it is the common hour format being used in Indonesia. To convert it to 12-hour format, you can use the following rules:

- If the hour is less than 12, it is AM (ante meridian) time.
- If the hour is 12, it is noon (12 PM).
- If the hour is greater than 12, subtract 12 to get the PM (post meridian) time.

:::

## Monthly Prayer Times

Monthly prayer times require less part compared to daily prayer times. You only need to provide the year and month.

```http title="HTTP GET"
https://api.myquran.com/v2/sholat/jadwal/{year}/{month}
```

:::warning

Monthly prayer times ONLY accept `yyyy/mm` or `yyyy/m` date format.
If you use `yyyy-mm` or `yyyy-m`, the API will return an 404 error. In this example, we also use `KOTA SEMARANG` and June 2025 as the parameter.

```json title="Error Response"
{
	"status": false,
	"error": 404,
	"message": "page not found",
	"request": {
		"path": "/sholat/jadwal/1433/2025-06"
	}
}
```

:::

### Valid Year and Month Format for Monthly Prayer Times

For year and month using `yyyy/mm` format, January is `01`, February is `02`, and so on. For example, June is `06`.

You can also use `yyyy/m` format, where January is `1`, February is `2`, and so on. For example, June is `6`.
